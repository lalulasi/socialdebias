# 论文—代码—实验覆盖核对报告

核对基准：`罗俊2241507.pdf`（100 页）与《SocialDebias 实验完整复现指南》
（36 页）。本次不核验数据内容、样本数量和论文表格中的既有数值，只核验方法、
参数、实验入口与指标口径。

## 结论

仓库原状态不能严格复现论文。主实验入口虽然基本齐全，但存在若干会改变方法或
指标含义的问题：旧模型少一层 128 维瓶颈；BERT 不是论文所写的全参数微调；
NLI Soft14 指南命令实际不产生软标签训练；消融脚本忽略对抗数据；NRC 特征被少量
手写词替代；Top-K、JS 与 IG baseline 口径不符；SocialDebias-Adv 三种子被错误地
做成多数投票。上述代码问题已修正。旧 checkpoint 仍可读取，但只能标记为 legacy，
不能继续作为修正后实现的论文结果。

## 已修正的关键偏差

1. 模型分支由旧版 `768→384→2` 改成论文所述 `768→384→128→2`。评估脚本会
   从权重形状识别旧 checkpoint，避免静默错载。
2. BERT 基线取消“冻结前 11 层”，改成标准全参数微调；补齐 weight decay 与
   label smoothing。原 BERT 表格必须重跑。
3. `nli_soft14` 不再依赖 `--use_contrastive` 才能执行。随机抽取某个改写版本时，
   其 `p_entail` 也随该版本一起抽取，不再错误地使用同一原文所有改写的均值。
4. 训练对抗数据补齐真正的 NLI 第三道过滤；概率索引从模型 `id2label` 解析，
   不再硬编码 0/1/2。
5. NRC 8 维改为读取正式词典、停用词过滤、累计后 L2 归一化。旧手写词实现只作为
   `legacy_seed_v0` 保留，用于读取历史 checkpoint。
6. IG 改为直接对 word embedding 做积分，baseline 为零向量；Top-K 按公式 (5.12)
   使用 `|交集|/K`；JS 对排序后的完整归因幅值分布做 Softmax，不要求共同词元。
7. SocialDebias-Adv 评估改用 `_bert_baseline.pt`，GossipCop/Weibo21 选择
   `surface_all`，并输出逐 seed、跨 seed mean/std 以及全 tone/source 数据集汇总。
8. 新增仓库内 `evaluate_endef_adversarial.py`，消除复现指南所称“随仓库提供”但原
   仓库实际缺文件的问题，并移除机器绝对路径。

## 论文文本自身仍需人工修订

这些问题无法仅靠改代码消除，否则会在论文不同段落之间任选其一：

1. §5.3 表 5-3 注释写“BERT F1=0.8703 < 0.85”，算术上错误。且 0.8703 是表中
   test F1，自适应机制应读取 validation F1。建议删掉该数字比较，改报三个 seed
   实际 `best_val_f1` 与触发结果。
2. §3.8.3 把自适应机制写成 epoch 后按验证 F1 动态调节，且称
   `lambda_bias_base=1.0`；§5.5.3 与复现指南则写成训练前“规模或 baseline val F1”
   双信号，基础值为 0.5。当前代码按后者实现，论文 §3.8.3 应同步重写。
3. 附录 A / 复现指南把偏置 MLP 写成 `768+8→384→128`，正文 §3.7.2 的门控公式
   则是先把 8 维投影到 768，再融合成 768 维后进入 MLP。代码按正文公式实现；附录
   应改为“门控输入 768+768，门控输出 768，偏置 MLP 768→384→128→2”。
4. §5.4.2 的 Top-K 公式是 `交集/K`，但 §5.4.4 又把表中数值称为 Jaccard，并且把
   0.1446 误写成“未微调 BERT”（表 5-8 中它属于 SocialDebias）。修正代码后表 5-8
   的三个指标必须重跑，旧数值不能沿用。
5. 表 5-7 标题与正文称“三方评估”，但 PolitiFact/GossipCop 没有 BERT 行。修正后的
   评估脚本会输出 BERT；论文应补行，或把这两个数据集明确改称“两方评估”。
6. 正文把零样本模型称为 DeepSeek-V4。代码现已把模型 ID 参数化并默认
   `deepseek-v4-flash`；正式复现时应在论文和结果文件中记录 API 实际返回/支持的
   精确模型版本，不能只写产品族名称。
