# SocialDebias 新一轮论文实验执行步骤

版本：paper-v2（基于论文—代码核对后的实现）  
固定随机种子：`42 / 2024 / 3407`  
适用目标：重新生成论文实验结果，再据实修改论文第 4、5、6 章及摘要。

## 1. 本轮实验的硬性原则

1. 旧 checkpoint 和旧表格数字全部视为 legacy，不与新结果混合求均值。
2. 所有神经网络主结果必须跑满三个随机种子；均值和标准差使用总体标准差
   （`numpy.std`，`ddof=0`）。
3. checkpoint 只按验证集 Macro-F1 选择，测试集只在选定 checkpoint 后报告，不能用
   test F1 选择配置。
4. 英文主干使用 `bert-base-uncased`，中文主干使用 `bert-base-chinese`；BERT 基线为
   全参数微调。
5. 统一训练参数：AdamW、`lr=2e-5`、`weight_decay=0.01`、
   `label_smoothing=0.1`、`max_length=512`、`epochs=3`；PolitiFact batch=4，
   GossipCop/Weibo21 batch=16。
6. SocialDebias 统一结构为 `768→384→128→2`，不得把旧 `768→384→2` checkpoint
   混入新实验。
7. 正式主模型在三个数据集上统一使用 `surface_all`，不再按数据集挑选不同“最优”
   配置。其组成固定为：GRL + Consist + 8 维 NRC + InfoNCE + NLI 双软标签 +
   Adaptive λ。
8. `surface`、`surface_adaptive`、17 维特征以及去组件版本只用于消融和机制分析。
9. 自适应 λ 是否触发必须读取本轮同 seed BERT 的 `best_val_f1` 和实际训练集规模，
   禁止在论文中预先写死 PolitiFact“不触发”。
10. 结果方向与旧论文不一致时，以新结果为准，修改论点，不为复现旧数字而调参。

## 6. P3：统一主模型 `surface_all`

三个数据集使用完全相同的算法开关。仅 batch、语言、NRC 词典和原始数据路径不同。

### 6.1 PolitiFact 与 GossipCop

```bash
for DS in politifact gossipcop; do
  BS=$([ "${DS}" = politifact ] && echo 4 || echo 16)
  for SEED in 42 2024 3407; do
    python scripts/train_socialdebias_surface.py \
      --dataset "${DS}" --language en --seed "${SEED}" \
      --epochs 3 --batch_size "${BS}" --max_length 512 \
      --lr 2e-5 --weight_decay 0.01 --label_smoothing 0.1 \
      --hidden_dim 384 --bottleneck_dim 128 \
      --lambda_fact 1.0 --lambda_bias 0.5 --lambda_consist 0.3 \
      --surface_feat_dim 8 --surface_lexicon_path "${NRC_EN}" \
      --adaptive_lambda --adaptive_size_thresh 1000 \
      --adaptive_f1_thresh 0.85 --adaptive_scale 0.1 \
      --use_contrastive --lambda_contrast 0.3 --temperature 0.07 \
      --use_soft_labels --alpha_floor 0.5 \
      --lambda_fact_soft 0.5 --soft_label_floor 0.5 \
      --orig_pkl "data/sheepdog/news_articles/${DS}_train.pkl" \
      --adv_pkl "data/qwen_adv/${DS}_p_entail_paper_v2.pkl" \
      --save_dir "${MODEL_DIR}" --save_suffix surface_all \
      2>&1 | tee "${LOG_DIR}/surface_all_${DS}_seed${SEED}.log"
  done
done
```

## 10. P7：六组件消融

完整模型加 6 个去组件变体，共 7 个配置。PolitiFact 和 GossipCop 各跑三 seed。
消融使用固定 λ，不启用 Adaptive λ，以保证每次只改变一个因素。

| 后缀 | 相对 Full 的唯一变化 |
|---|---|
| `abl_full` | 全组件 |
| `abl_no_grl` | `--lambda_bias 0` |
| `abl_no_infonce` | 移除 `--use_contrastive`，保留分类软标签 |
| `abl_no_consist` | `--lambda_consist 0` |
| `abl_no_surface` | `--surface_feat_dim 0`，不传 NRC |
| `abl_no_advaug` | 移除 paired 数据、InfoNCE、NLI 分类软标签 |
| `abl_no_labelsmooth` | `--label_smoothing 0 --lambda_fact_soft 0`，仍保留加权 InfoNCE |

Full 命令模板如下，其他变体严格按上表只做对应改动：

```bash
python scripts/train_ablation.py \
  --dataset gossipcop --language en --seed 42 \
  --epochs 3 --batch_size 16 --max_length 512 \
  --lr 2e-5 --weight_decay 0.01 --label_smoothing 0.1 \
  --hidden_dim 384 --bottleneck_dim 128 \
  --lambda_fact 1.0 --lambda_bias 0.5 --lambda_consist 0.3 \
  --surface_feat_dim 8 --surface_lexicon_path "${NRC_EN}" \
  --use_contrastive --lambda_contrast 0.3 --temperature 0.07 \
  --use_soft_labels --alpha_floor 0.5 \
  --lambda_fact_soft 0.5 --soft_label_floor 0.5 \
  --orig_pkl data/sheepdog/news_articles/gossipcop_train.pkl \
  --adv_pkl data/qwen_adv/gossipcop_p_entail_paper_v2.pkl \
  --save_dir "${MODEL_DIR}" --save_suffix abl_full
```

每个 checkpoint 随后执行：

```bash
python scripts/evaluate_ablation_adv.py \
  --dataset gossipcop --language en --seed 42 \
  --save_suffix abl_full --variants A,B,C,D \
  --surface_lexicon_path "${NRC_EN}" \
  --ckpt "${MODEL_DIR}/socialdebias_gossipcop_en_seed42_abl_full.pt" \
  --output_dir "${RUN_ROOT}/ablation_adv"
```

将数据集、seed 和 suffix 扩展为完整矩阵。验收数量：`2 数据集 × 7 配置 × 3 seed = 42`
个训练 history，以及 42 个对抗评估 JSON。表 5-10 可以主报 GossipCop，PolitiFact 放
附录，但正文中对每个组件的结论必须与两数据集实际结果一致。

## 12. P9：NLI 机制实验

旧论文对 `floor=0.5` 与 Soft14 的开关描述不够清楚。本轮采用可审计的两因素设计：

| 配置 | InfoNCE 的 NLI 权重 | 分类软标签 | floor |
|---|---:|---:|---:|
| `nli_hard` | 否 | 否 | — |
| `nli_contrast_only` | 是 | 否 | 0.5 |
| `nli_cls_only` | 否 | 是 | 0.5 |
| `nli_dual_floor0` | 是 | 是 | 0.0 |
| `nli_soft14` | 是 | 是 | 0.5 |

统一保留 InfoNCE 本身；`nli_hard` 只是不用 `p_entail` 加权。五种配置均在三个数据集、
三个 seed 上运行，共 45 次。命令以 P3 为模板，但本阶段不启用 Adaptive λ：

- `nli_hard`：保留 `--use_contrastive`，移除 `--use_soft_labels`，设
  `--lambda_fact_soft 0`。
- `nli_contrast_only`：使用 `--use_contrastive --use_soft_labels --alpha_floor 0.5`，
  设 `--lambda_fact_soft 0`。
- `nli_cls_only`：移除 `--use_contrastive`，保留
  `--lambda_fact_soft 0.5 --soft_label_floor 0.5`。
- `nli_dual_floor0`：使用全部双机制，设
  `--alpha_floor 0 --soft_label_floor 0`。
- `nli_soft14`：使用全部双机制，设
  `--alpha_floor 0.5 --soft_label_floor 0.5`。

每个配置使用独立 `--save_suffix`。论文表 5-12 要按这五个可验证定义重写；不得继续沿用
旧的 36 次实验说法。
